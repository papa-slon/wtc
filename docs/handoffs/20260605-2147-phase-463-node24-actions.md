## Scope
This phase migrates the active GitHub Actions workflow off Node 20-runtime official actions after GitHub emitted
deprecation annotations on the green post-merge `main` runs. It does not change bot runtime behavior, secrets, production
deployment, database state, provider probes, or live controls.

## Findings
1. Severity P1 - GitHub Actions warnings on `main` were caused by official JavaScript actions still running on Node 20,
   not by the WTC project runtime. Evidence: the workflow already set `node-version: 24`, while CI annotations named
   `actions/checkout@v4`, `actions/setup-node@v4`, and `actions/upload-artifact@v4`. Recommendation: update action majors
   rather than add a temporary opt-out. Target part: `.github/workflows/ci.yml`.
2. Severity P1 - GitHub will begin running actions on Node 24 by default on June 16, 2026, so leaving the workflow on
   Node 20-runtime action majors would make the forced switch unproven. Evidence: GitHub changelog. Recommendation: prove
   Node 24-runtime action majors now through PR and post-merge `main` CI. Target part: CI release gate.
3. Severity P2 - Historical docs mention earlier `actions/*@v4` runs and should remain as evidence of those older runs.
   Recommendation: update only active truth/status docs and the active workflow. Target part: documentation integrity.

## Decisions
1. Updated `actions/checkout@v4` to `actions/checkout@v6` in both jobs.
2. Updated `actions/setup-node@v4` to `actions/setup-node@v6` in both jobs, retaining `node-version: 24` and `cache: npm`.
3. Updated `actions/upload-artifact@v4` to `actions/upload-artifact@v7` for reviewed visual evidence manifests.
4. Did not add `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24`, because version migration is the cleaner durable path.
5. Did not add `ACTIONS_ALLOW_USE_UNSECURE_NODE_VERSION`, because it is an insecure/deprecated fallback.

## Risks
1. PR/main CI must prove the exact workflow commit; local checks cannot execute GitHub-hosted action runtimes.
2. The artifact action major changed, so the e2e job's manifest upload must be observed on GitHub.
3. This CI migration does not clear production deploy/source gates.
