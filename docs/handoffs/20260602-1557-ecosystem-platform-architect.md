## Findings
1. Severity: High. The existing preflight summary-root guard is lexical, not symlink-hard. It rejects absolute/UNC/URL/traversal/non-`logs/` strings and checks `relative(ROOT, absoluteRoot)`, but it never calls `realpath` or `lstat`; `writePreflightSummary()` then creates the directory and writes a summary below the candidate path, which would follow an existing `logs/...` symlink or Windows junction to an outside target. Evidence: `scripts/preflight-log-root.mjs:1`-`2`; `scripts/preflight-log-root.mjs:4`; `scripts/preflight-log-root.mjs:12`-`28`; `scripts/preflight-log-root.mjs:36`-`40`. Recommendation: add a realpath/lstat confinement layer before summary writes; keep the public `resolvePreflightLogRoot()` contract and update only its internals unless implementation needs an exported low-level helper. Target part: preflight log-root helper.

2. Severity: High. The retained artifact scanners have the same escape class, but with different policy semantics. `scan-lms-db-e2e-artifacts.mjs` and `check-retained-visual-artifacts.mjs` both validate user paths lexically with `resolve()`/`relative()`, then walk/read with `statSync()` and `readFileSync()`; `statSync()` follows symlinks, so a workspace-local path string can still scan or read outside the workspace through a symlink/junction. Evidence: `scripts/scan-lms-db-e2e-artifacts.mjs:126`-`140`; `scripts/scan-lms-db-e2e-artifacts.mjs:193`-`200`; `scripts/scan-lms-db-e2e-artifacts.mjs:262`-`273`; `scripts/check-retained-visual-artifacts.mjs:63`-`74`; `scripts/check-retained-visual-artifacts.mjs:96`-`103`; `scripts/check-retained-visual-artifacts.mjs:221`-`224`. Recommendation: switch scanner walking and manifest/OCR sidecar reads to an `lstat`-first guard and reject symlink/reparse entries or follow them only after `realpath` proves containment. Target part: artifact scanner filesystem boundary.

3. Severity: Medium. Do not replace the scanner path policies with the preflight `logs/` policy. Preflight summary roots must stay under `logs/`, but the LMS artifact scanner intentionally scans `test-results`, `playwright-report`, `tests/e2e/screenshots`, and `logs/lms-db-e2e`, while the visual checker defaults to `tests/e2e/screenshots`. Evidence: `scripts/preflight-log-root.mjs:20`-`21`; `scripts/scan-lms-db-e2e-artifacts.mjs:6`; `scripts/check-retained-visual-artifacts.mjs:6`; `docs/IMPLEMENTED_FILES.md:85`-`96`; `docs/IMPLEMENTED_FILES.md:28`-`35`. Recommendation: share only the low-level realpath containment primitive; keep scanner-local rules for allowed roots, missing explicit roots, dynamic marker manifests, visual manifests, and OCR sidecars. Target part: shared helper vs scanner-local boundary.

4. Severity: Medium. A script-level helper is the right sharing layer; a new `@wtc/*` package or product runtime abstraction would be over-scoped. The monorepo rules keep product/business logic in packages, but this phase is operator evidence tooling under `scripts/`; the five summary-writing preflights already share `preflight-log-root.mjs`, and package scripts expose these as opt-in acceptance commands. Evidence: `AGENTS.md:74`-`88`; `docs/handoffs/0000-orchestrator-seed.md:123`-`124`; `package.json:30`-`37`; `scripts/lms-s3-r2-live-preflight.mjs:9`; `scripts/lms-external-scanner-live-preflight.mjs:8`; `scripts/billing-stripe-webhook-replay-preflight.mjs:23`; `scripts/billing-stripe-checkout-preflight.mjs:9`; `scripts/axioma-handoff-preflight.mjs:20`. Recommendation: implement a small `scripts/workspace-path-guard.mjs` or equivalent script-only helper for `realpathSync.native`/`lstatSync` checks, then consume it from `preflight-log-root.mjs` plus the two scanners. Target part: script utility boundary.

5. Severity: Medium. Windows junctions are the main cross-platform edge case. Current tests cover Windows absolute paths, POSIX absolute paths, UNC strings, URL roots, traversal, and non-`logs/` roots, but they do not create a directory symlink/junction under `logs/`; a junction can preserve the lexical `logs/...` prefix while resolving outside the workspace. Evidence: `tests/integration/preflight-log-root.test.ts:41`-`54`; `tests/integration/preflight-log-root-wiring.test.ts:70`-`84`; `tests/integration/preflight-log-root-wiring.test.ts:86`-`108`. Recommendation: compare canonical real paths with `path.relative(realWorkspaceRoot, realCandidate)` rather than string prefix checks; use `fs.symlinkSync(target, link, process.platform === 'win32' ? 'junction' : 'dir')` in tests, with safe cleanup and a skip/fallback only if the OS refuses link creation. Target part: cross-platform regression coverage.

6. Severity: Low. The opt-in preflight command boundary should remain unchanged. Existing tests assert LMS object storage, LMS external scanner, Stripe checkout, Stripe webhook replay, and Axioma handoff preflights are opt-in and outside default gates, then scan the redacted summary roots. Evidence: `tests/integration/lms-object-storage-live-preflight.test.ts:35`-`45`; `tests/integration/lms-external-scanner-live-preflight.test.ts:33`-`43`; `tests/integration/billing-stripe-checkout-preflight.test.ts:21`-`31`; `tests/integration/billing-stripe-webhook-replay-preflight.test.ts:21`-`31`; `tests/integration/axioma-handoff-preflight.test.ts:32`-`42`. Recommendation: keep implementation to helper/scanner/test changes; do not run live preflights or add these commands to default gates. Target part: acceptance command boundary.

7. Severity: Low. `audit-append-only-role-preflight.mjs` is not a log-root writer and should not be pulled into this phase beyond retained-artifact scanner coverage. It validates a credentialed Postgres URL and writes one audit row after explicit acceptance, but it does not use `resolvePreflightLogRoot()` or write a summary root. Evidence: `scripts/audit-append-only-role-preflight.mjs:10`-`13`; `scripts/audit-append-only-role-preflight.mjs:73`-`80`; `package.json:35`-`36`. Recommendation: leave audit-role live DB acceptance out of Phase 3.57 unless a separate credentialed acceptance phase is started. Target part: out-of-scope preflight.

## Decisions
Minimal recommended implementation shape:

1. Add one script-local realpath/lstat primitive, either as `scripts/workspace-path-guard.mjs` or a clearly named internal helper if the implementer chooses not to add a new file. Do not add a product package.
2. The primitive should cache `realWorkspaceRoot = realpathSync.native(process.cwd())`, use `lstatSync()` before following entries, and use `realpathSync.native()` plus `path.relative()`/`path.isAbsolute()` to decide whether a resolved target remains inside the real workspace root.
3. For preflight summary writes, keep `resolvePreflightLogRoot()` as the logs-only policy owner. Because target roots may not exist yet, validate existing ancestors first, create the directory, then realpath-check the final root immediately before writing `summary-*.json`.
4. For scanners, keep `resolveWorkspacePath()` policy local to each scanner, but call the shared realpath guard for explicit roots, dynamic marker manifests, visual review manifests, OCR sidecars, and every walked entry. Prefer rejecting symlink/reparse entries in evidence roots rather than silently following them.
5. Preserve current no-value-echo behavior: refusal messages should name the rejected class (`artifact roots must be workspace-local paths`, `dynamic marker manifest rejected`, `visual review manifest rejected`, or similar) without printing the raw symlink target, secret-like outside path, or matched artifact content.
6. Do not run live provider preflights, preview, Playwright, DB mutation, SSH/nginx/systemd, bot services, CI, deploy, or production monitoring in this phase.

## Risks
1. TOCTOU remains possible if a local adversary swaps a directory for a symlink between check and write/read. For this operator tooling slice, checking immediately before each summary write and each scanner read is a pragmatic boundary; stronger open-by-handle/no-follow logic would be over-scoped and less portable.
2. `realpathSync.native()` cannot resolve a non-existent final write root. The implementation must guard existing ancestors before `mkdirSync()` and re-check the final root after creation.
3. Windows path casing, drive letters, UNC forms, and junctions make string-prefix comparisons unsafe. Use `path.relative()` on canonical real paths and treat an absolute or `..` relative result as outside.
4. Strictly rejecting symlink/reparse entries may reject a future operator's symlinked evidence directory even when it points inside the workspace. That is acceptable for retained evidence: a normal copied/generated directory is clearer and avoids ambiguous archive provenance.
