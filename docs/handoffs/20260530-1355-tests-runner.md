## Decisions

1. **No duplication** of tortila-mapping.test.ts, tv-atomic-grant-revoke.test.ts, or
   migration-0003-pglite.test.ts — these are already fully covered by
   `packages/bot-adapters/src/__tests__/tortila-mapping.test.ts` (35 tests) and
   `tests/integration/db-0003.test.ts` (14 tests). Creating stub files was not done.

2. **BW-005** is tested at the repo layer (createManualReviewItem + audit). The route-handler
   guard (when `event.userId` is null → return 200 + no call to applyStripeEvent) is tested
   indirectly by verifying no `billing.webhook_received` audit row exists for the phantom event.

3. **BW-006** concurrent duplicate via Promise.all tests the read-then-write idempotency guard
   in `applyStripeEvent`. PGlite is single-threaded so both calls serialize; the invariant of
   exactly one audit row is always satisfied. The test correctly documents that ≥1 applied=true
   is the minimum guarantee under serialization.

4. **ADM-RBAC-001..003** are pure logic tests (no PGlite needed for the RBAC guard itself) as
   specified in the test matrix.

5. The lint gate is RED due to pre-existing errors in non-owned files. My files introduce 0
   new lint errors.

## Risks

1. **Lint gate failure** from pre-existing errors: `apps/web/src/app/admin/system-health/page.tsx`
   (fmtDate unused) and `packages/bot-adapters/src/http.ts` (instanceId unused). These need
   to be fixed by the owning agents before the gate turns green.

2. **Real-PG gate NOT RUN**: no `DATABASE_URL` / `REAL_POSTGRES_DATABASE_URL` / Docker.
   PGlite covers the repo logic but `db:migrate` / `db:seed` / real connection-pool tests remain
   unverified.

3. **BW-006 PGlite concurrency note**: PGlite is single-threaded in-process. The concurrent
   duplicate test serializes naturally. Under real Postgres with multiple connections, the
   TOCTOU race (SELECT-then-INSERT in applyStripeEvent) could still allow two concurrent
   inserts. The `insertWebhookEventOnce` repo uses a DB-level UNIQUE constraint (ON CONFLICT DO
   NOTHING) and is the correct idempotency guard for production.
