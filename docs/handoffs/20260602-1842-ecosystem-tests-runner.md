## Decisions
- This pass did not run the four requested gates; it audited readiness and failure risks only.
- Use current status/blocker docs over stale local logs when reporting Phase 3.62 gate state.
- Keep default e2e separate from full/local build gates because Playwright owns a dev server lifecycle.
- Keep safe preview separate from default e2e because it binds port 3000 as an interactive browser preview stream.
- Keep LMS DB browser acceptance, live object-store/scanner, Stripe, Axioma, DB rollout, SSH/nginx/systemd, bot services/control, GitHub CI, deploy, and production monitoring outside this site-readiness gate audit.
## Risks
- Root `npm test` may expose current Vitest/PGlite regressions not covered by recent focused tests; do not infer root-suite health from Phase 3.61 focused audit-runner Vitest.
- Web build can fail independently of TypeScript/lint if Next route generation, middleware, app-router server components, or runtime imports changed since the last current build.
- Default e2e writes screenshots and may generate traces/reports on failure; any retained visual/binary artifacts need separate review before archive.
- Running default e2e while another dev server is active on `localhost:3100` should fail because `reuseExistingServer` is false; clear stale listeners rather than reusing them.
- Running safe preview in parallel with e2e can confuse evidence and port expectations (`3000` versus `3100`); run it last and stop it cleanly after manual smoke.
- A safe-preview redacted stream lowers leak risk but does not make raw terminal buffers, preview logs, screenshots, videos, or traces archive-safe.

- RUN in this audit: protocol/status/script/config/log inspection only; `git rev-parse --show-toplevel` as a read-only repo-state check.
