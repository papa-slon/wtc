## Decisions
- Treated Phase 3.15 local DB-backed file/embed storage as the current baseline, not as production upload readiness.
- Did not run full gates or Playwright because this request is pre-implementation discovery and explicitly says not to run expensive full gates now.
- Focused recommended tests on small, local, evidence-producing slices that can run before any public production upload rollout.
## Risks
- Some production-readiness primitives already exist in `@wtc/lms`, but without tests, persistence, and route gating they can be mistaken for completed production scan/quarantine/retention support.
- Default e2e remains valuable for navigation/mobile smoke, but it cannot prove DB-backed upload/download behavior while it runs without `DATABASE_URL`.
- If object storage is added without static guards, storage calls could drift into React pages or thin route adapters instead of staying behind package/repository boundaries.
