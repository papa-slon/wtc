**DEFERRED (each with a confirmed, still-live blocker — unanimous across auditors):**
- `lessons.embed_html` / writable `'embed'` — no server-side HTML sanitizer exists (stored-XSS gate). `'embed'` is a
  forward-compat CHECK value only; never selectable, never rendered as raw HTML.
- `materials` file-meta (`file_key`/`file_name`/`file_size_bytes`/`mime_type`) — upload security review BLOCKED (ROADMAP §7); no object storage.
- `pinned_links owner_type='global'` — non-additive DROP+ADD CHECK (drizzle-kit emits a 2nd constraint, leaving both live) + Q-6 bundling undecided.
- `courses.slug` — no slug-URL routing consumer (routes use `[courseId]` UUID); adding it = dead schema.
- `lesson_progress.state` — `deriveLessonState` still works; no video-scrub consumer; column would be dual-truth/dead.
- Teacher community / pinned-link / teacher-profile web surfaces — repos exist, no UI; their own follow-up slice.

## Cross-auditor conflicts resolved (operator decisions)

1. **ContentType union width.** education/frontend leaned `'video'|'article'|'link'`; db-architect/tests noted the DB CHECK
   includes `'embed'`. **Decision: widen the TS union to all four** (`video|embed|article|link`) so render switches stay
   exhaustive and match the column, but **exclude `'embed'` from the Zod write enum and every editor selector** — the type
   documents the future, the Zod enum is the gate. (Matches the security lane's M-4.)
2. **Retire vs keep `deriveContentType`.** Unanimous retire (dual-truth); operator **deleted** the function + its unit test
   rather than leaving a deprecated stub (no consumer after backfill).
3. **`updateLessonAction`.** Not in the operator's original scope; education-implementer showed the content_type write path
   for existing lessons would otherwise be dead (create-only). **Decision: add it** (CSRF-first pipeline) — folded into scope.
4. **Student catalogue layer.** `/app/education` runs on the older Phase-1.7 `lmsService` (thin), not `features/lms`. Because
   `lib/lms-types.ts` aliases its `CourseView`/`LessonView` to the `@wtc/db` DTOs, the new DTO fields flow through for free —
   so the catalogue shows the level badge + tags **without** switching layers (switching would empty it in demo + risk the e2e).

## Findings → fixes (summary)

- **(security M-1, pre-existing XSS) `materialSchema.url` + `lessonSchema.videoUrl` used `z.url()` without `.startsWith('https://')`** —
  a `javascript:`/`data:` URL passed validation and rendered as a clickable href. **Fixed** for material url, videoUrl, and the
  new externalUrl (all `.url().startsWith('https://')`), plus a render-time `safeHttpsUrl()` guard on every outbound href.
- **(db-architect) backfill UPDATE** hand-appended to the generated 0005 (drizzle-kit can't emit it); verified on PGlite against the real SQL.
- **(education) missing `updateLessonAction`** — added (the content_type write path for existing lessons).
- **(all) `deriveContentType` dual-truth** — retired at all callsites + deleted; the column is authoritative.
- **(self-caught during gates)** 3 test-harness issues fixed before green: the PGlite test needed `seedDatabase` (roles FK);
  a static guard matched my own "deriveContentType retired" comment (comment reworded); the canonical LMS-actions count
  10→11 for the new action.

## Decisions

1. Bounded slice = 4 additive columns; all 6 deferred items keep a confirmed blocker (ADR-021).
2. `deriveContentType` retired + deleted (co-landed with the migration; no dual-truth).
3. `'embed'` is a forward-compat CHECK value only — never written, never rendered as raw HTML (security gate).
4. All URL write paths https-only + render-time `safeHttpsUrl` guard (closes a pre-existing gap).
5. Added `updateLessonAction`; kept the student catalogue on `lmsService` (DTO alias carries the new fields).
6. No new table — 41 tables; backfill UPDATE hand-edited into 0005.

## Risks

- **Demo/e2e coverage.** The teacher editor + DB-backed detail are not reachable by the seeded e2e (demo user isn't a
  teacher; no `DATABASE_URL` in e2e) — those paths are covered by PGlite + static guards. The catalogue level badge IS e2e-checked.
- **PGlite ≠ real Postgres.** CHECK enforcement + array operators are unreliable on PGlite (B1 NOT RUN); the Zod enum is the
  binding validator and the migration must be applied to a real `wtc_test` to prove the CHECK/backfill on Postgres.
- Still NOT production-ready (B1/B2/B4/B6 unchanged).
